작업규칙

용어정리

  • 웹 (World Wide Web) : 인터넷에 연결된 사용자들이 서로의 정보를 공유할 수 있는 공간을 의미한다.
  • HTML (Hyper Text Markup Language) : 웹을 통해 보여지는 모든 문서와 사이트를 이루는 기본적인 언어이다.
  • CSS (Cascading Style Sheet) : HTML로 만들어진 문서의 텍스트 색상이나 크기, 이미지 크기나 위치, 표의 색상, 배치방법 등 웹 문서의 디자인 요소를 담당하는 언어이다.
  • 태그 (Tag) : HTML 문서를 구성하는 명령어를 말한다.
  • 엘리먼트 (Element) : HTML 문서를 구성하는 요소를 의미한다. 일반적으로 시작 태그, 내용, 종료 태그로 구성된다.
  • 어트리뷰트 (Attribute) : 엘리먼트에 부여할 수 있는 특성을 의미한다. 기본값이 설정되어 있으나 어트리뷰트를 선언하여 다른 값으로 설정할 수 있다.
  • 선택자 (Selector) : 엘리먼트에 CSS 스타일을 적용하기 위한 패턴이다.
  • 속성 (Property) : 엘리먼트에 부여할 CSS 스타일 특성을 의미한다.
  • 속성값 (Value) : 속성에 부여할 논리적인 값을 말한다.

네이밍

기본

  • 케밥 케이스(Kebab Case) : 아이디(id) / 클래스(class)

    변수를 만들 때, 각 단어를 구분하기 위해서 대시(-)를 사용하고 문자는 모두 소문자로만 표기한다.

    ※ 단어와 숫자를 조합할 경우, 단어 뒤에 숫자(한자리정수)로 표기

  • 카멜 케이스(Camel Case) : 함수(function) / 변수(var)

    변수를 만들 때, 각 단어를 구분하기 위해서 첫번째 단어를 제외한 각 단어의 첫문자를 대문자로 표기한다.

    ※ 단어와 숫자를 조합할 경우, 단어 뒤에 숫자(한자리정수)로 표기

  • 스네이크 케이스(Snake Case) : 그 외 파일명, 폴더명 등

    변수를 만들 때, 각 단어를 구분하기 위해서 언더스코어(언더라인, 언더바 _ )를 사용하고 문자는 모두 소문자로만 표기한다.

    ※ 단어와 숫자를 조합할 경우, 단어 뒤에 숫자(한자리정수)로 표기

정의 예제
아이디(id) / 클래스(class) bg-color1 ~ bg-color9
함수(function) / 변수(var) bgColor1 ~ bgColor9
그 외 파일명, 폴더명 등 bg_color1 ~ bg_color9

HTML

프로그램 또는 스토리보드의 화면ID를 기본으로 한다. 특별한 이유로 ID를 가져올 수 없는 경우, 다음의 순서를 따른다.

  • 프로젝트에서 요구하는 특정한 HTML 규칙을 우선한다.
  • 그룹핑숫자_메뉴이름_의미_상태 순으로 조합한다.
  • 파일의 특징, 분류, 성격을 알 수 있도록 네이밍한다.
예제 설명
FIS_SR_010.html 스트로보드 화면ID
201.html 그룹핑숫자
popup_news.html 특징, 분류, 성격

ID

<label> 태그처럼 관련 폼요소와 매핑을 위해 임의의 id를 생성하는데 id의 형식은 “UI-형태/의미”로 조합하고 페이지내 중복이 없도록 맨뒤에 숫자(한자리정수)를 붙인다.

<label for="UI-형태/의미1">input 타이틀</label><input type="text" id="UI-형태/의미1">
<input type="radio" id="UI-형태/의미2"><label for="UI-형태/의미2">radio 타이틀</label>

※ 개발 작업시 폼요소의 id가 변경되는 경우, 필히 기존에 매핑되어져있던 label(for값)도 동일한 네이밍으로 바꿔줘야 한다.

CLASS

  • 타입-형태-의미 순으로 조합하여 작성한다.
  • 하이픈(-)은 2개 이상의 단어를 조합할 때만 사용한다.
  • 하이픈(-)을 3개 이상의 단어를 초과하여 사용하는 것은 지양한다.
예) list-dot
  • 다중 클래스는 최대 3개 이상은 사용하지 않도록 주의하고 하나의 명확한 단어로만 네이밍을 정의한다.
예) .board.multi
  • 범주에 따라서 네이밍을 구분하여 사용한다.
-area / -box (or) -list / -panel (or) -case / -item (or) -clip

※ -wrap은 큰 레이아웃에만 사용하고 콘텐츠에서는 사용하지 않는다. (예: wrap, page-wrap, popup-wrap, tooltip-wrap 등)

IMAGE

  • 예약어_형태_의미_상태 순으로 조합하고 예약어를 근거하여 작성한다.
  • 배너나 사진처럼 비트맵 이미지일 경우 JPG 포맷으로 제작(기본 Maximum 퀄리티 세팅)하고 벡터 이미지일 경우 GIF 포맷으로 제작한다.
  • 투명이나 그라데이션 요소의 이미지일 경우 PNG 파일로 제작(png24 포맷)한다.
  • 같은 분류의 이미지가 두개 이상이면 파일이름 마지막에 숫자(한자리정수)를 추가하여 구분한다.
  • 불필요한 배경을 줄여 용량을 최소화하고 여백만을 위한 이미지 사용은 금하며 CSS 사용을 지향한다.
구분 예약어
배경 bg_*
버튼 btn_*
아이콘 ico_*
탭메뉴 tab_*
주메뉴 gnb_*
측면메뉴 snb_*
배너 bnr_*
스프라이트 spr_*
사진 img_*
섬네일 thumb_*
임시 temp_*

※ 이미지명에는 '@'문자 사용불가

CSS

공통 스타일시트를 제외하고 모든 스타일시트는 업무 또는 서비스명을 따른다.

SCRIPT

  • 추가 js파일을 작성하게 될 경우에는 기능, 업무코드, 서비스명을 따른다.
  • 활성화가 필요한 경우, 비활성화 상태의 화면을 우선 작업하고 활성화 시 네이밍을 on 클래스로 동작한다.

HTML

HTML은 W3C validation을 통과해야 한다.

문서타입(DTD)

  • 브라우저에게 웹 페이지의 문서 종류를 알려주는 선언문으로, 해당 문서의 최상단에 선언한다.
  • DTD는 브라우저가 올바른 화면표시(Rendering)를 위해 반드시 필요하다.
  • 올바른 DTD를 사용하면 브라우저가 표준에 맞춰 렌더링을 하기 때문에 다양한 환경에서 상호 운용성을 보장할 수 있다.

문서타입 선언

모든 HTML 페이지 시작 지점에 공백없이 HTML5 문서타입을 선언한다.

<!DOCTYPE html>

언어(Language)

문서 루트인 html 요소에 언어(lang) 속성을 추가한다.

<html lang="ko">

※ 국문 ko, 영문 en, 중문 zh, 일문 ja

HEAD

<meta>, <title>, <link>, <style>, <script> 순으로 엘리먼트를 나열한다.

메타(Meta)

  • 문자열 인코딩(Encoding)을 명시적으로 선언하되 DB의 인코딩 방식과도 관련이 있으므로 반드시 담당 개발자와 협의하여 정해야 한다.
<meta charset="utf-8">

※ 다국어 utf-8, 완성형 euc-kr

  • 모바일 기기에서 자동 전화걸기를 방지할 경우에는 아래 내용을 선언한다.
<meta name="format-detection" content="telephone=no">
  • 최신모드로 지정된 DOCTYPE에 상관없이 IE8 이상 버전에서 항상 최신 표준 모드로 렌더링한다.
<meta http-equiv="X-UA-Compatible" content="IE=Edge">
  • 뷰포트(Viewport)는 개발과 관련하여 특별한 사항이 없는 경우 device-width와 사용자 확대 불가로 선언한다.
<meta name="viewport" content="width=device-width,initial-scale=1.0,maximum-scale=1.0,minimum-scale=1.0,user-scalable=no">

그 외 메타(Meta)

  • 제작자의 이름
<meta name="author" content=“제작자 이름">
  • 저작권에 관한 정보
<meta name="copyright" content="">
  • 검색엔진에서 검색되는 키워드
<meta name="Keywords" content="html,tag,mata,">
  • 60초마다 새로고침
<meta http-equiv="refresh" content="60">
  • 주소로 5초 후 이동
<meta http-equiv="refresh" content="5" url="주소">
  • 캐쉬가 되지 않게
<meta http-equiv="Pragma" content="no-cache">
  • 캐쉬 만료(파기)일
<meta http-equiv="expires" content="mon, 08 sep 2003 10:10:10 Gmt">
  • 웹문서의 설명글
<meta name="description" content="웹퍼블리셔를 위한 사이트 입니다.">
  • 검색엔진 허가
<meta name="robots" content="all">

타이틀(Title)

  • 현재 페이지의 제목을 선언한다.
  • 특별한 사항이 없는 경우, 서비스명 혹은 브랜드명은 가장 마지막에 작성한다.
  • 태그와 코멘트 사이에 공백을 넣지 않는다.
<title>활성화된 탭명 | 현재 페이지명 | 카테고리명 | 브랜드명</title>

링크(Link)

  • type, rel, href 순서로 어트리뷰트를 선언한다.
<link type="text/css" rel="stylesheet" href="">

※ HTML5 DTD일 경우, type은 생략 가능하다.

  • 파비콘
<link rel="shortcut icon" href="favicon.ico">

스타일시트(Style sheet)

  • type 어트리뷰트를 선언한다.
  • 하위에 선언되는 style은 1depth 들여쓴다.
<style type="text/css">
	…
</style>

※ HTML5 DTD일 경우, type은 생략 가능하다.

스크립트(Script)

  • type, src 순서로 어트리뷰트를 선언한다.
  • 하위에 선언되는 script는 1depth 들여쓴다.
  • 스크립트 선언 위치는 <head> 내에 선언하는 것을 원칙으로 하되, 성능 상의 이슈 및 특별한 사항이 있는 경우 선언위치를 변경할 수 있다.
<script type="text/javascript" src=""></script>

<script type="text/javascript">
	…
</script>

※ HTML5 DTD일 경우, type은 생략 가능하다.

일반규칙

닫기 태그

XML에서는 필수이지만 HTML5에서는 닫기태그( / )를 사용하지 않아도 무방하다.
예외적으로, 작업 시 공통 규칙이거나 as-is 소스를 기반으로 할 경우에는 상황에 맞추서 작성한다.

=== 잘못된 예 ===
<meta ... />
<link ... />
<lnput ... />
<lmg ... />
<br ... />
<hr ... />
<col ... />

=== 올바른 예 ===
<meta ... >
<link ... >
<lnput ... >
<lmg ... >
<br ... >
<hr ... >
<col ... >

영문 소문자

DTD를 제외한 모든 엘리먼트와 어트리뷰트는 소문자로 작성한다.

=== 잘못된 예 ===
<SPAN Class="" ID=""></SPAN>
<a href="" onClick="">

=== 올바른 예 ===
<span class="" id=""></span>
<a href="" onclick="">

어트리뷰트(Attribute)

어트리뷰트는 태그의 기본 속성순으로 작성한다.

<a href="" class="">
<img src="" alt="" style="">
<input type="" placeholder="" title="" id="" class="" style="">

어트리뷰트값(Value)

어트리뷰트값에는 항상 겹따옴표("")를 사용한다.

=== 잘못된 예 ===
<img src='' width='' height='' alt=''>

=== 올바른 예 ===
<img src="" width="" height="" alt="">

엔티티(Entity) 코드

  • 특수기호는 엔티티 코드를 변환하여 사용한다.
  • <, >, ", ', & 등의 특수기호를 엔티티 코드로 변환하지 않으면, 브라우저가 이를 시작/종료 엘리먼트나 어트리뷰트로 잘못 해석 할 수 있다.
  • 엔티티 코드는 ISO-8859-1을 기준으로 하며, 다음 경로에서 확인 할 수 있다.

아이디(ID)

  • 아이디 선택자는 한 문서에 유일하게 한번만 사용해야 한다.
  • 레이아웃을 제외한 아이디는 스타일을 지정하지 않는다.

클래스(Class)

  • 클래스는 문서에서 여러 번 사용 할 수 있다.

이미지

  • 의미없는 이미지인 경우 alt값은 공백으로 처리한다.
  • 내용이 많은 경우 히든 텍스트로 작성한다.

그 외 사항

  • 엘리먼트은 모듈 단위로 마크업 되어야 한다.
  • 의미에 맞도록 시맨틱한 마크업을 지향하고 중첩 태그의 사용을 지양한다.
  • 표현을 위한 태그 사용은 지양하고 표현에 해당하는 사항은 CSS에 작성한다.
    <font>, <b>, <u> ...
  • HTML 저장 시 불필요한 공백문자는 제거한다.

주석

  • 불필요한 주석은 쓰지 않으며, 상태변화가 필요한 부분은 비 개발자도 이해할 수 있도록 자세히 표기한다.
  • 아이디, 클래스 명으로 기능을 유추할 수 있으면 주석을 달지 않는다.
  • 소스 보기 등의 방법으로 개발 시스템 정보, 개발자 정보 등을 유추할 수 없어야 한다.
  • 단, 불명확한 아이디, 클래스 명이라면 역할을 주석으로 기입한다.
  • 시작과 종료의 주석 내용은 동일해야 한다.
  • 주석기호와 코멘트 사이에 공백을 넣어주고 닫기 주석과 코멘트 사이에는 공백없이 작성한다. 
=== 잘못된 예 ===
<!--주석-->
<!--//주석--> (or) <!-- // 주석 -->

=== 올바른 예 ===
<!-- 주석 -->
<!-- //주석 -->

수정주석

  • 주석을 기입하는 경우 해당 수정날짜로 시작하는데 여러 줄일 경우 시작과 끝의 범위로 표시하고 한 줄인 경우 해당 라인의 마지막에 표시한다.

단일 라인

<strong>…</strong> <!-- 0000-00-00 수정 -->

복합 라인

<!-- s:0000-00-00 수정 -->
<div class="div">
	…
</div>
<!-- e:0000-00-00 수정 -->

중복 라인

<strong>…</strong> <!-- 0000-00-00 수정1 --> <!-- 0000-00-00 수정2 -->

<!-- s:0000-00-00 수정2 -->
<!-- s:0000-00-00 수정1 -->
<div class="div">
	…
</div>
<!-- e:0000-00-00 수정1 -->
<!-- e:0000-00-00 수정2 -->

개발주석

  • 개발에 전달되어야 하는 주석은 [D]로 시작하고 뒤에 전달할 내용을 작성한다.
<ul>
	<li class="on"></li> <!-- [D] 활성화 시 on클래스 처리 -->
	<li></li>
</ul>

CSS

CSS는 사용 가능한 Hack과 CSS3를 제외하고 W3C validation을 통과해야 한다.

일반규칙

문서의 첫 줄에는 인코딩(Encoding)을 선언하며 인코딩은 HTML과 동일한 인코딩을 선언한다.

@charset "utf-8";

파일 불러오기

  • CSS 데이터를 로딩할 경우 @import로 호출하거나 공통선택자(*)를 자주 사용하면 웹 페이지의 성능 저하를 가져오기 때문에 가급적 사용하지 않는다.
    ※ XP IE8 조건의 경우는 CSS 줄이 2~3000을 넘게 되면 렌더링 오류가 발생하므로 그런 경우에만 @import를 사용하여 별도 호출한다.
  • CSS 데이터는 <head> 상단에 공통부분과 콘텐츠 부분으로, 간단한 페이지의 경우 하나의 CSS 파일로 로딩될 수 있도록 한다.
<link type="text/css" rel="stylesheet" href="">

※ HTML5 DTD일 경우, type은 생략 가능하다.

영문 소문자

속성 및 속성값은 모두 소문자로만 작성한다.

=== 잘못된 예 ===
BORDER:1px solid #FBE3E4;BACKGROUND-COLOR:#E5ECF9;COLOR:#CC0000;

=== 올바른 예 ===
border:1px solid #fbe3e4;background-color:#e5ecf9;color:#cc0000;

나열

속성은 특별한 경우를 제외하고는 줄내림 없이 한줄로 나열하여 작성한다.

=== 잘못된 예 ===
display:block;
margin:0;
font-size:12px;

=== 올바른 예 ===
display:block;margin:0;font-size:12px;

공백

선택자와 괄호 사이는 공백을 넣어주고 괄호{} 안의 내용은 공백을 최소화 한다.

=== 잘못된 예 ===
.selector{property: value; property: value;}

=== 올바른 예 ===
.selector {property:value;property:value;}

자식 선택자 또는 근접 선택자를 사용할 때는 선택자 사이에 공백을 준다.

=== 잘못된 예 ===
.selector>p { ... }
.selector+span { ... }

=== 올바른 예 ===
.selector > p { ... }
.selector + span { ... }

괄호() 안에서는 쉼표(,) 뒤에 공백을 넣지 않고 최소화 한다.

=== 잘못된 예 ===
color:rgba(0, 0, 0, .5);

=== 올바른 예 ===
color:rgba(0,0,0,.5);

!important는 속성값 사이에 공백을 준다.

=== 잘못된 예 ===
margin:0!important;

=== 올바른 예 ===
margin:0 !important;

따옴표

속성 선택자에는 겹따옴표("")를 사용한다.

=== 잘못된 예 ===
[type='text'] { ... }
[attribute^='selector'] { ... }
[attribute*='selector'] { ... }

=== 올바른 예 ===
[type="text"] { ... }
[attribute^="selector"] { ... }
[attribute*="selector"] { ... }

속성값에는 홑따옴표('')를 사용한다.

=== 잘못된 예 ===
{font-family:"돋움";}
{content:"";}

=== 올바른 예 ===
{font-family:'돋움';}
{content:'';}

이미지의 속성값에는 따옴표를 사용하지 않는다.

=== 잘못된 예 ===
{background:url('ex.png');} (or) {background:url("ex.png");}

=== 올바른 예 ===
{background:url(ex.png);}

줄바꿈

복수 선택자를 한줄로 할 경우, 코드변경 시 의도치않게 누락되는 위험성이 있으므로 쉼표(,) 뒤에서 줄바꿈 한다.

=== 잘못된 예 ===
.selector1, .selector2 { ... }

=== 올바른 예 ===
.selector1,
.selector2 { ... }

빈줄

의미있는 객체를 구분하기 위하여 코드 그룹 간 1줄씩 빈 줄을 넣어 준다. 단, 빈줄의 간격은 1줄을 초과하지 않는다.

=== 잘못된 예 ===
/* 주석1 */
.selector1 { ... }


/* 주석2 */
.selector2 { ... }

=== 올바른 예 ===
/* 주석1 */
.selector1 { ... }

/* 주석2 */
.selector2 { ... }

0(Zero)

  • 값이 0일 경우는 단위를 생략한다.
  • 소수점 값이 0으로 시작할 경우는 0을 생략한다.
=== 잘못된 예 ===
margin:0px;
padding:0px 10px;
font-size:0.8em;

=== 올바른 예 ===
margin:0;
padding:0 10px;
font-size:.8em;

축약(Shorthand)

축약 가능한 값은 축약한다.

=== 잘못된 예 ===
font-style:italic;font-variant:small-caps;font-size:16px;line-height:1.5em;font-weight:bold;font-family:Gulim,Arial,sans-serif;
background-color:#fff;background-image:url(경로/파일명);background-repeat:no-repeat;background-position:left center;background-size:100% auto;
list-style-type:square;list-style-position:inside;list-style-image:url(경로/파일명);
margin:25px 15px 30px 15px;
padding:15px 15px 15px 15px;
border:none;
color:#ffffff; (or) color:#ff4400;

=== 올바른 예 ===
font:italic small-caps bold 16px/1.5em Gulim,Arial,sans-serif;
background:#fff url(경로/파일명) no-repeat 0 50%;background-size:100% auto;
list-style:square inside url(경로/파일명);
margin:25px 15px 30px;
padding:15px;
border:0;
color:#fff; (or) color:#f40;

Z-INDEX

  • z-index 수치 간격은 10단위로 적용한다.
  • 페이지 단위에서 1000을 넘는 경우를 만들지 않는다.
  • 팝업 레이어의 경우는 1000부터 시작하고 알럿의 경우는 10000으로 한다.

배경(Background)

배경 속성에 색상을 추가할 경우 색상을 앞쪽에 위치한다.

=== 잘못된 예 ===
background:url(경로/파일명) no-repeat 0 0 #fff;

=== 올바른 예 ===
background:#fff url(경로/파일명) no-repeat 0 0;

폰트(Font)

영문폰트만 선언할 경우 특정 브라우저에서 폰트를 올바르게 출력하지 못하는 경우가 있으므로 한글 폰트 선언 시 한글, 영문 폰트를 모두 선언한다.

=== 잘못된 예 ===
font-family:'돋움'; (or) font-family:dotum;

=== 올바른 예 ===
font-family:'돋움',dotum;

들여쓰기

CSS 코드를 작성할 때는 들여쓰기를 하지 않는다. 단, 미디어쿼리나 애니메이션과 같이 중괄호 {}가 중첩되는 경우는 예외로 한다.

@media all and (min-width: 480px) {
	.selector {width:100%;}
}

미디어 쿼리(Media Query)

미디어쿼리는 관련 규칙이 있는 위치에 보기 쉽도록 모아 작성한다.

.selector { ... }
.selector-avatar { ... }
.selector-selected { ... }
@media (min-width: 640px) {
	.selector { ... }
	.selector-avatar { ... }
	.selector-selected { ... }
}

벤더 프리픽스(Vendor Prefix)

벤더 프리픽스(공급사 접두어)는 일반 프로퍼티보다 먼저 선언한다.

※ 주요 웹 브라우저 공급자가 새로운 실험적인 기능을 제공할 때 이전 버전의 웹 브라우저에 그 사실을 알려주기 위해 사용하는 접두사(prefix)를 의미한다.

=== 잘못된 예 ===
box-shadow:0 1px 2px rgba(0,0,0,.5);-webkit-box-shadow:0 1px 2px rgba(0,0,0,.5);

=== 올바른 예 ===
-webkit-box-shadow:0 1px 2px rgba(0,0,0,.5);box-shadow:0 1px 2px rgba(0,0,0,.5);

그 외 사항

  • 줄여 쓸 수 있는 속성들(shorthand properties)은 줄여 쓰고 풀어 쓸 때는 줄여 쓸 때의 순서를 따른다.
    단, 줄여 쓰는 경우 관련된 속성이 초기화되어 상속되지 않으므로 주의한다.
  • CSS는 모듈화를 기본으로 하며, 모듈간의 독립성을 유지한다.
  • HTML 엘리먼트에 인라인타입(style)을 직접 기입하지 않는다. 단, 개발 시 필요한 경우에만 부분적으로 기입한다.
  • 크로스 브라우징을 위해 제시된 핵(hack)에 한해 최소한의 조건으로 사용을 허용한다.

주석문

큰 주석문의 경우는 아래와 같이 작성하고

/* ====================
    주석
==================== */

기본적인 주석문은 간단한 형식으로 작성한다.

/* 주석 */

주석

  • 주석은 간결하게 작성한다.
  • 주석기호와 코멘트 사이에 공백을 넣어준다.
=== 잘못된 예 ===
/*주석*/

=== 올바른 예 ===
/* 주석 */

수정주석

주석을 기입하는 경우 해당 수정날짜로 시작하는데 여러 줄일 경우 시작과 끝의 범위로 표시하고 한 줄인 경우 해당 라인의 바로 윗줄에 표시한다.

단일 라인

/* 0000-00-00 수정 */
.selector { … }

복합 라인

/* s:0000-00-00 수정 */
.selector1 { … }
.selector2 { … }
.selector3 { … }
/* e:0000-00-00 수정 */

중복 라인

/* 0000-00-00 수정2 */
/* 0000-00-00 수정1 */
.selector { … }

/* s:0000-00-00 수정2 */
/* s:0000-00-00 수정1 */
.selector1 { … }
.selector2 { … }
.selector3 { … }
/* e:0000-00-00 수정1 */
/* e:0000-00-00 수정2 */

SCRIPT

  • 함수
    • 함수명을 정의할 때는 카멜표기법을 준용한다.
    • 업무적인 특성 혹은 적용해야 하는 함수의 의미를 고려하여 적절한 함수명을 정의한다.
  • 변수
    • 한 줄에는 하나의 변수만을 선언한다.
    • 변수명에는 기호나 특수문자를 사용하지 않는다.
    • 주요 기능순으로(권장) 같은 기능은 그룹을 짓도록 한다.
  • 주석
    • 스크립트 내에서 흐름이나 사용을 설명할 필요가 있을 경우나 영역을 구분지어 가독성을 높일 경우에 사용한다.
    • 한줄( // ) 주석의 경우 설명이 필요할 때 사용한다.
    • 여러 줄( /* */ ) 주석의 경우 해당 부분의 영역, 이벤트, 함수 등의 의미를 구분 짓고자 할 때 사용한다.
