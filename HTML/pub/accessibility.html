모바일 기기마다 음성 안내를 위한 애플리케이션을 지원한다.

  • Voice Assistant (Android)
    • [실행] 설정 > 접근성 > Voice Assistant > 실행
    • [단축키] 설정 > 접근성 > 볼륨 키 단축키 탭 > 볼륨 키 단축키 설정 (볼륨 키를 3초 동안 눌러 접근성 도구 실행)
  • VoiceOver (iOS)
    • [실행] 설정 > 손쉬운 사용 > VoiceOver > 실행
    • [단축키] 설정 > 손쉬운 사용 단축키 > VoiceOver 선택 (종료키를 연속 3번 누르면 바로 실행)

role 속성

속성 내용
role="main" 본문의 주요 콘텐츠 영역으로 한 페이지 내에 1개만 사용이 가능하며, <article>, <aside>, <footer> 요소의 하위 요소로 사용할 수 없다.
role="navigation" 다른 페이지 또는 페이지 내 특정 영역으로 이동하는 링크 콘텐츠 영역으로 주로 메인 메뉴 및 서브 메뉴 등에 사용할 수 있다.
role="banner" 비슷한 의미로 <header> 요소를 사용할 수 있으나 <header role=""banner">로 사용하였다면 한 페이지에서 한 개의 <header> 요소만 사용하길 권장한다.
role="form" 폼과 관련된 요소의 모임을 표현하는 영역으로 서버에 전송될 수 있는 콘텐츠를 포함 할 수 있다.
role="heading" 헤딩 (예: role="heading" aria-level="1")
role="button" 버튼
role="img" 이미지
role="text" 문장
role="tablist|presentation|tab|tabpanel" 탭 목록, 탭, 탭 패널
role="tooltip" 툴팁
role="alert" 알럿
role="alertdialog" 알럿 대화상자
role="dialog" 대화상자
role="search" 검색
role="radiogroup" / role="radio" aria-checked="true|false" 라디오 그룹 / 라디오
role="checkboxgroup" / role="checkbox" aria-checked="true|false" 체크박스 그룹 / 체크박스
role="progressbar" 프로그래스바 (예: role="progressbar" aria-valuenow="75" aria-valuemin="0" aria-valuemax="100")

속성 내용
aria-hidden="true|false|undefined" 숨김 상태

aria-expanded="true|false|undefined" 확장 상태
aria-selected="true|false|undefined" 선택 상태 (button에는 사용불가)
aria-label="string" 간결한 설명
aria-labelledby="ID reference list" 간결한 설명 참조
aria-describedby="ID reference list" 자세한 설명 참조
aria-controls="ID reference list" 제어 대상
aria-haspopup="true" 팝업 안내
aria-modal="true|false" 모달
aria-live="polite|assertive|off" 실시간
aria-atomic="true|false" 업데이트 영역
aria-disabled="true|false" 비활성화
aria-multiselectable="true|false" 하나 이상의 요소 선택
aria-current="page" 현재 페이지 강조

aria-current="step" 현재 단계 강조
aria-current="time" 현재 시간 강조
aria-required="true" 해당 항목이 필수 항목(Require)임을 알 수 있도록 제공

지침기준

한국형 웹 콘텐츠 접근성 지침 2.1

http://www.wah.or.kr/Participation/guide.asp

원칙 (4) 지침 (13) 검사항목 (24)
1. 인식의 용이성 1.1 대체 텍스트 1.1.1. (적절한 대체 텍스트 제공) 텍스트 아닌 콘텐츠는 그 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야 한다.
1.2 멀티미디어 대체 수단 1.2.1. (자막 제공) 멀티미디어 콘텐츠에는 자막, 대본 또는 수화를 제공해야 한다.
1.3. 명료성 1.3.1. (색에 무관한 콘텐츠 인식) 콘텐츠는 색에 관계없이 인식될 수 있어야 한다.
1.3.2. (명확한 지시 사항 제공) 지시 사항은 모양, 크기, 위치, 방향, 색, 소리 등에 관계없이 인식될 수 있어야 한다.
1.3.3. (텍스트 콘텐츠의 명도 대비) 텍스트 콘텐츠와 배경 간의 명도 대비는 4.5 대 1 이상이어야 한다.
1.3.4. (자동 재생 금지) 자동으로 소리가 재생되지 않아야 한다.
1.3.5. (콘텐츠 간의 구분) 이웃한 콘텐츠는 구별될 수 있어야 한다.
2. 운용의 용이성 2.1 입력장치 접근성 2.1.1 (키보드 사용 보장) 모든 기능은 키보드만으로도 사용할 수 있어야 한다.
2.1.2. (초점 이동) 키보드에 의한 초점은 논리적으로 이동해야 하며 시각적으로 구별할 수 있어야 한다.
2.1.3. (조작 가능) 사용자 입력 및 컨트롤은 조작 가능하도록 제공되어야 한다.
2.2. 충분한 시간 제공 2.2.1. (응답시간 조절) 시간제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 한다.
2.2.2. (정지 기능 제공) 자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있어야 한다.
2.3. 광과민성 발작 예방 2.3.1. (깜빡임과 번쩍임 사용 제한) 초당 3~50 회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 한다.
2.4. 쉬운 내비게이션 2.4.1. (반복 영역 건너뛰기) 콘텐츠의 반복되는 영역은 건너뛸 수 있어야 한다.
2.4.2. (제목 제공) 페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 한다.
2.4.3. (적절한 링크 텍스트) 링크 텍스트는 용도나 목적을 이해할 수 있도록 제공해야 한다.
3. 이해의 용이성 3.1. 가독성 3.1.1. (기본 언어 표시) 주로 사용하는 언어를 명시해야 한다.
3.2. 예측 가능성 3.2.1. (사용자 요구에 따른 실행) 사용자가 의도하지 않은 기능(새 창, 초점에 의한 맥락 변화 등)은 실행되지 않아야 한다.
3.3. 콘텐츠의 논리성 3.3.1. (콘텐츠의 선형 구조) 콘텐츠는 논리적인 순서로 제공해야 한다.
3.3.2. (표의 구성) 표는 이해하기 쉽게 구성해야 한다.
3.4. 입력 도움 3.4.1. (레이블 제공) 사용자 입력에는 대응하는 레이블을 제공해야 한다.
3.4.2. (오류 정정) 입력 오류를 정정할 수 있는 방법을 제공해야 한다.
